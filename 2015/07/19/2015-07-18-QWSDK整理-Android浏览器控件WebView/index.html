QWSDK整理-Android浏览器控件WebView

发表于 2015-07-19   |  

QWSDK本身通过加载本地html来实现SDK界面显示，SDK内部加载html代码比较复杂，是前上司王辉所写，这里总结一下webview的基本的要点

##WebView基本实现

//需要为webview设置 WebViewClient

//setWebChromeClient,设置Chrome处理

WebView myWebView = (WebView) findViewById(R.id.webview); 
webview.setWebViewClient(new WebViewClient(){
//一般WebViewClient需要重载的方法，使得打开新网页时在webview中打开而不是在系统浏览器中打开
@Override
public boolean shouldOverrideUrlLoading(WebView view, String url) {
       view.loadUrl(url);//webview打开网页的方法
       //QWSDK中使用loadDataWithBaseURL方法加载本地html内容
       //view.loadDataWithBaseURL("", htmlString, "text/html", "utf-8", "");
        return true;
 }
 //可根据需要重载onPageStarted、 onPageFinished等方法
});

##Android方法与js的互相调用
首先设置webview为允许js

webview.getSettings().setJavaScriptEnabled(true);

  • js调用Android方法：

    webview.addJavascriptInterface(new MyJavaScriptInterface(param..),”js2java”);
    //Android方法写在QWJavaScriptInterface类中，如:
    public void myClose(String colseString){…}
    这样在js中，可以使用windows.js2java.myClose(“colseString”);来调用myClose方法

  • Android方法调用js

    view.loadUrl(url);//载入html

然后可通过点击html控件调用js，或者可以使用以下方式直接调用html中的js:

webview.loadUrl("javascript:js_fun()");

若程序采用proguard混淆，还需要注意在混淆配置文件中加入
-keep public class com.example.zhengjt.MyJavaScriptInterface{
public ;
}
来防止java的js接口类被混淆，避免js找不到java方法

另外SDK中更新html界面的方式是使用runOnUiThread，如果当前线程是UI线程,那么行动是立即执行。如果当前线程不是UI线程,操作是发布到事件队列的UI线程。
