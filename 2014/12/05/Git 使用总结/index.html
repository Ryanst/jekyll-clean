Git 使用总结

Posted on 2014-12-05   |  

版本回退:

  • 回退到上num版本

bash git reset --hard HEAD~num
num的值表示要往上回退num个版本

  • 回退到上一个版本，zsh中^需要转义，需要使用git reset –hard HEAD\^
    bash git reset --hard HEAD^

  • 回退到特定的版本
    bash git reset --hard commitId
    commitId可以通过git log来得到，可以写版本号前几个字符

  • 如果回滚版本后反悔了，又想恢复到新版本，但是找不到新版本的版本号，可以
    bash git reflog
    这个命令记录了你的每一次命令

操作回退：

  • 放弃当前修改，回到上一次git 命令后的初始状态(这里—符号后一定要有一个空格)
    bash git checkout — filename
  • 把当前add到暂存区的file回退到工作区(相当于取消add这个file的操作)
    bash git reset HEAD filename

远程操作:

bash git remote add origin git@github.com:Ryanst/test.git
在本地目录下键入这个命令，将远程库与本地该目录关联

关联后，使用命令git push -u origin master第一次推送master分支的所有内容

分支管理

  • 常用命令

查看分支：git branch

创建分支：git branch

切换分支：git checkout

创建+切换分支：git checkout -b

合并某分支到当前分支：git merge

删除分支：git branch -d

  • git解决冲突的一般方法：

bash git merge mybranch //然后修改冲突文件 git add filename git commit -m “解决冲突" git branch -d mybranch

使用用git log –graph命令可以看到分支合并图可以看到分支合并图

储存为commit的当前工作区现场：
当手头工作没有完成时，先把工作现场git stash一下，然后去修复bug，修复后，再git stash pop，回到工作现场。

多人协作

  • 查看远程库信息
    bash git remote -v；
    本地新建的分支如果不推送到远程，对其他人就是不可见的；

  • 从本地推送分支
    bash git push origin branch-name
    如果推送失败，先用git pull抓取远程的新提交；

  • 在本地创建和远程分支对应的分支
    bash git checkout -b branch-name origin/branch-name

  • 建立本地分支和远程分支的关联
    bash git branch --set-upstream branch-name origin/branch-name；

从远程抓取分支，使用git pull，如果有冲突，要先处理冲突
